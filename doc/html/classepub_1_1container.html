Detailed Description

The EPUB container file hierarchy.

XHTML and SVG documents added are parsed for metadata information. Those meta elements that have names prefixed with epub: are loaded into the file_metadata map associated with the file. Currently, the supported tags are:

  • epub:spine
    Controls inclusion of the item in the Spine (reading order) of the publication. Items not in the spine are automatically omitted from the table of contents.
    • include (default for XHTML)
    • omit (default for SVG)
  • epub:toc
    Controls inclusion of the item in the Table of Contents.
    • include (default, but see above)
    • omit
  • epub:properties
    A space-separated list of manifest item properties controlling a content document. See https://www.w3.org/TR/epub-33/#sec-item-resource-properties for a list of common properties. Common properties are svg for XHTML with embedded SVG and scripted for pages with embedded JavaScript.

For SVG documents, metadata is freeform and can be a mixture of character data and elements with no prescribed structure. For locating the relevant elements, the parser will assume that all meta elements from the namespace "http://www.w3.org/1999/xhtml" in the top-level SVG metadata element are candidates. The name prefix is the same as XHTML.
